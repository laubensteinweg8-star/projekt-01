# Mini-Lexikon: Terminal- und Git-Befehle

## Terminal-Befehle

### `pwd`
Zeigt an, in welchem Ordner du dich gerade befindest.

### `ls -l`
Listet alle Dateien und Ordner im aktuellen Verzeichnis auf, mit Details.

### `cd <ordner>`
Wechselt in einen anderen Ordner.

### `touch <dateiname>`
Erstellt eine neue, leere Datei.

### `nano <dateiname>`
Öffnet eine Datei im Editor, um sie zu bearbeiten.

### `cat <dateiname>`
Zeigt den Inhalt einer Datei im Terminal an.

---

## Git-Befehle

### `git status`
Zeigt den aktuellen Zustand des Repositories:
- ob Dateien geändert wurden
- ob etwas zum Commit bereit ist
- ob alles sauber ist

### `git add <datei>`
Markiert eine Datei als „bereit für den Commit“.

### `git commit -m "Nachricht"`
Speichert die Änderungen dauerhaft im lokalen Repository.

### `git push`
Überträgt die Commits zu GitHub.

### `git log --oneline`
Zeigt die Commit-Historie in kurzer Form.

### `git show <commit>:<datei>`
Zeigt den Inhalt einer Datei aus einem früheren Commit.
